# Jurist AI вЂ” Refinements Plan

## Overview

This document details three major refinements to the Jurist AI platform:

1. **Professional Summary в†’ Backend-Generated** (currently frontend chat hack)
2. **Executive Summary в†’ Backend-Generated** (same issue, same fix)
3. **Report Generation в†’ Complete Overhaul** (currently basic, needs to be impressive)

---

## Problem Analysis

### Current State: Professional Summary & Executive Summary

**Location:** `frontend/src/App.jsx` lines 602вЂ“634 (`fetchNarrativeVerdict`)

Currently, both the "Professional Summary" (hero card on OverviewPage) and the "Executive Summary" (bottom-left card on OverviewPage) display the **same** `narrativeVerdict` string. This string is generated by:

1. Frontend sends a hardcoded prompt to the **chat endpoint** (`POST /chat`)
2. The chat endpoint treats it like a regular user message вЂ” no access to structured analysis signals
3. The LLM only sees the raw document text stored in the chat session, **not** the actual analysis results (severity scores, risk breakdown, confidence levels, NLP features, position weights, etc.)

**Problems:**
- The summary is generated **without** the rich structured signals the backend already computed
- It's a "chat hack" вЂ” abusing the chat system instead of having a dedicated endpoint
- The same generic text is shown in two different UI locations with different labels
- No structured data вЂ” just a plain text blob with no actionable metadata
- Inconsistent quality вЂ” the LLM improvises because it lacks the actual numbers

### Current State: Report Generation

**Location:** `frontend/src/components/ReportsPage.jsx` (290 lines)

The current report is essentially a **re-layout of existing dashboard data** with zero additional intelligence:

- 4 stat boxes (score, total clauses, flagged, AI deep scan %)
- The same `narrativeVerdict` text as "Executive Verdict"
- A severity bar chart (same as dashboard)
- Category breakdown bars (same as dashboard)
- Top 10 risky clauses with explanations (same data as clauses page)
- Copy-to-clipboard produces a bare-bones text dump
- Print produces an unstyled browser print

**Problems:**
- No new insights beyond what's already visible on other pages
- No actionable recommendations per category
- No compliance assessment (GDPR, CCPA, etc.)
- No risk scoring methodology explanation
- No "what to negotiate" or "red lines" section
- Copy/export is a plain text dump, not a structured report
- No table of contents, section numbering, or professional formatting
- No timestamp, document fingerprint, or audit trail metadata

## Key Design Decisions

1. **Summaries are generated at analysis time** (not on-demand) вЂ” so they're instant when the user views results. The report is generated on-demand since it's heavier.

2. **Two distinct summaries** вЂ” Professional Summary (2-3 sentences, hero card) and Executive Summary (4-6 sentences, detail card) serve different purposes and will have different content.

3. **Report generation is a separate LLM call** вЂ” it produces category deep dives, compliance checks, and action plans that are NOT part of the base analysis. This keeps the core analysis fast while the report is a premium output.

4. **Structured JSON from LLM** вЂ” both summaries and report sections are returned as structured JSON, not plain text blobs. This allows the frontend to render them with rich formatting.

5. **Caching** вЂ” generated reports are cached in the DB so they're not re-generated on every page visit.
